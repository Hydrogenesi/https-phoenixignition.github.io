All contributions follow the Phoenix Quantumonix flow.

---

## 1. Start With a Tension

Open an issue using the **Tension Card** template:

- Context
- Desired resolution
- Scale band
- Canon relationship

No PR may be opened without a tension.

---

## 2. Select Operators

Maintainers and contributors map the tension to:

- TUA operators
- Canon 6.0 constraints
- Required badges
- Ceremony scope

Operators define the transformation.

---

## 3. Run the Ceremony

Use the `scripts/ceremony_runner` to perform:

- **Call** — declare ceremony ID
- **Witness** — roles: initiator, witness, steward
- **Transform** — apply operators
- **Seal** — badges, lineage, logs

The ceremony runner will generate:

- `PHX-QX-YYYY-MM-DD-CVN.md`
- Updated badges
- Updated lineage metadata

---

## 4. Open the PR

A valid PR must include:

- Tension card reference
- Operator list
- Ceremony log
- Updated badges
- Origin + Canon + Operator + Ceremony metadata

CI will reject PRs missing any of these.

---

## 5. Merge Conditions

A PR may merge only if:

- Continuity is intact
- All badges are present
- Lineage is declared
- Ceremony log is complete
- Canon constraints are satisfied

Silent edits are not allowed.

---

## 6. Remixing

Remixes must declare:

- Parent artifacts
- Remix type (translation, contextualization, extension, fork)
- Ceremony reference
- Canon relationship

Remixes without lineage are rejected.

---

## 7. Governance

The CVN governance layer uses:

- TUA tension law
- Phoenix operator logic
- Canon 6.0 scaling
- Quantumonix ceremonies

This ensures trauma-informed, reversible, auditable change.
