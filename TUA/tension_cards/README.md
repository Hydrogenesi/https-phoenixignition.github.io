This directory contains tension cards — the entry point for all changes in the
CVN·A_TUA_M system. A tension card defines a need, gap, or transformation that
initiates the Phoenix Quantumonix ceremony flow.

## Naming Convention

`TENSION-<YYYY-MM-DD>-<short-name>.md`

Example: `TENSION-2026-03-03-onboarding-gap.md`

## Required Fields

Each tension card must include:

- **Context:** What is the current state?
- **Desired resolution:** What is the target state?
- **Scale band:** At what scale does this tension operate?
- **Canon relationship:** How does this relate to Canon 6.0?
- **Operator candidates:** Which TUA operators may address this tension?

## Required Metadata

Every tension card must include the canonical header (see `CANON/canon_6.0_reference.md`).
